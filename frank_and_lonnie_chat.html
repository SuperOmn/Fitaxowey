Roadmap

Phase 1 — Git + GitHub Foundation

Before touching any infrastructure, get version control solid.

  • Create a public repo, practice branching, committing, PRs — terminal only
  • Docs: git-scm.com/doc
  • Docs: docs.github.com/en/get-started
Phase 2 — Docker Fundamentals

Learn containers in isolation before adding a database.

  • Run basic containers, understand images, volumes, networking
  • Docs: docs.docker.com/get-started
Phase 3 — SQL Basics

Learn CRUD and schema design with a local DB (PostgreSQL recommended).

  • Use dummy football data to practice — no real data yet
  • Docs: postgresql.org/docs/current
Phase 4 — Docker + SQL Together

Run a PostgreSQL instance inside Docker. Connect to it, run queries. This is where the project environment actually takes shape.

Phase 5 — AWS Data Ingestion

Find a football dataset, pull it in, load it into your Dockerized DB.

  • AWS Open Data Registry: registry.opendata.aws
  • Research required: find the specific dataset (sport, year, format)
Phase 6 — Iterate

Query real data, build out schema, document everything in the repo. Sports betting pivot lives here if you want it.
